【MCU】【CH32】基于CH32F103C8T6离线固件库和MDK环境下的工程模板搭建（纯手动）

本文最后更新于 2025-11-18T02:09:38+08:00

一、简介

CH32F103C8T6 是南京沁恒微电子（WCH）推出的一款 32 位微控制器，兼容 STM32F103C8T6（意法半导体的经典型号），采用 ARM Cortex-M3 内核，性能与成本平衡，适合替代 STM32 在各类嵌入式项目中的应用。以下是其核心特性和应用场景：

（1）核心参数

  • 内核与性能

    <ul>
	<li>32 位 ARM Cortex-M3 内核，最高 72MHz 主频，支持 Thumb-2 指令集。</li>
	<li>1.25DMIPS/MHz，内置硬件乘法器和除法器。</li>
</ul>
</li>
<li>
<p><strong>存储与外设</strong></p>

<ul>
	<li>64KB Flash（支持 ICP/IAP）、20KB SRAM。</li>
	<li>丰富通信接口：3 个 USART、2 个 SPI、2 个 I2C、1 个 USB 2.0 全速（12Mbps）、CAN 2.0B。</li>
	<li>1 个 12 位 ADC（16 通道，最高 1MHz 采样率）、2 个 12 位 DAC。</li>
	<li>7 个定时器（包括高级定时器），支持 PWM 输出和编码器接口。</li>
</ul>
</li>
<li>
<p><strong>工作条件</strong></p>

<ul>
	<li>供电电压：2.0V~3.6V，工作温度：-40&deg;C~+85&deg;C（工业级）。</li>
	<li>低功耗模式：睡眠、停止、待机。</li>
</ul>
</li>

（2）与 STM32F103C8T6 的兼容性

  • 硬件引脚兼容：完全兼容 STM32F103C8T6 的 LQFP48 封装，可直接替换。
  • 软件兼容：支持标准库（需使用 WCH 提供的固件库）和 HAL 库，大部分 STM32 代码可无缝移植。
  • 工具链支持：
    • 开发环境：Keil MDK、IAR、GCC（如 OpenOCD）。
    • 调试接口：支持 SWD/JTAG（与 STM32 相同）。

（3）应用场景

  • 工业控制：电机控制、传感器数据采集、PLC 模块。
  • 消费电子：智能家居设备、手持设备、无线充电。
  • 通信设备：串口转 USB/CAN 转换器、网关。
  • 替代方案：在 STM32 缺货或成本敏感场景中作为直接替代品。

（4）优势与注意事项

  • 优势

    <ul>
	<li><strong>成本更低</strong>：价格通常低于 STM32F103C8T6。</li>
	<li><strong>国产支持</strong>：供货稳定，技术支持响应快。</li>
	<li><strong>扩展特性</strong>：部分型号增加了 USB 和 CAN 等接口（如 CH32F103xC 系列）。</li>
</ul>
</li>
<li>
<p><strong>注意事项</strong></p>

<ul>
	<li><strong>固件库差异</strong>：虽然 API 与 STM32 类似，但细节可能不同，需参考 WCH 官方文档。</li>
	<li><strong>生态资源较少</strong>：第三方库和教程不如 STM32 丰富。</li>
	<li><strong>性能微调</strong>：部分外设（如 ADC）的精度和稳定性可能略逊于 STM32。</li>
</ul>
</li>

（5）开发资源

  • 官方资源： 
    <ul>
	<li>沁恒官网（<a data-link-icon="https://csdnimg.cn/release/blog_editor_html/release2.3.9/ckeditor/plugins/CsdnLink/icons/icon-default.png?t=P4F5" data-link-title="www.wch.cn" href="http://www.wch.cn/" title="www.wch.cn">www.wch.cn</a>）提供固件库、数据手册和开发工具（如 WCH-Link）。</li>
	<li>支持标准的 ARM 开发工具链（如 Keil、GCC）。</li>
</ul>
</li>
<li><strong>社区支持</strong>：国内开源社区（如 STM32 论坛）有部分 CH32 移植经验分享。</li>

/********************************** (C) COPYRIGHT *******************************

#include "debug.h"

/* Global define */


/* Global Variable */ 


/*******************************************************************************
* Function Name  : GPIO_Toggle_INIT
* Description    : Initializes GPIOA.0
* Input          : None
* Return         : None
*******************************************************************************/
void GPIO_Toggle_INIT(void)
{
	GPIO_InitTypeDef  GPIO_InitStructure;

	RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOC,ENABLE);	    
	GPIO_InitStructure.GPIO_Pin = GPIO_Pin_13;              
	GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_PP;           
	GPIO_InitStructure.GPIO_Speed=GPIO_Speed_50MHz;
	GPIO_Init(GPIOC, &GPIO_InitStructure);                  
}

/*******************************************************************************
* Function Name  : main
* Description    : Main program.
* Input          : None
* Return         : None
*******************************************************************************/
int main(void)
{
	NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2);
	Delay_Init();
	//USART_Printf_Init(115200);
	GPIO_Toggle_INIT();
	//printf("SystemClk:%d\r\n",SystemCoreClock);
	//printf("GPIO Toggle TEST\r\n");
    
	while(1)
	{
		Delay_Ms(1000);
		GPIO_WriteBit(GPIOC, GPIO_Pin_13, Bit_RESET);
		Delay_Ms(1000);
		GPIO_WriteBit(GPIOC, GPIO_Pin_13, Bit_SET);
	}
}
